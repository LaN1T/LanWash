# Покупка абонементов клиентом

## Статус

Утверждено. Готов к написанию плана реализации.

## Контекст

В текущей версии приложения абонементы — это back-office фича: админ вручную создаёт запись в `subscriptions` для конкретного пользователя, а при записи на мойку бэкенд автоматически списывает одну мойку (`paidPrice = 0`).

У клиента в мобильном приложении нет возможности ни просматривать, ни тем более покупать абонементы. Существующий экран `SubscriptionScreen` («Мои абонементы») не подключен к навигации.

## Цель

Дать клиенту возможность самостоятельно покупать абонементы через отдельный пункт бокового меню. Реализация в демо-режиме: реальная оплата не проводится.

## Требования

### Виды абонементов

1. **Готовый абонемент** — выбирается из предустановленных планов.
2. **Персональный абонемент** — клиент сам выбирает тип мойки, дополнительные услуги и количество моек, цена рассчитывается автоматически.

### Готовые планы

| Код | Название | Тип | Параметры | Скидка |
|-----|----------|-----|-----------|--------|
| `chistulya` | Чистюля | package | 5 моек | 10% |
| `blesk-master` | Блеск-мастер | package | 10 моек | 15% |
| `bezlimitka` | Безлимитка | unlimited | 30 дней | — |

Для готовых пакетов клиент дополнительно выбирает **тип мойки** (один из четырёх). Цена зависит от выбранного типа.

Для плана «Безлимитка» выбирается один тип мойки, на который действует безлимит в течение 30 дней.

### Персональный абонемент

Клиент последовательно выбирает:
1. Тип мойки (1 из 4).
2. Дополнительные услуги (множественный выбор).
3. Количество моек.

Объёмная скидка:
- 5 моек — 5%
- 10 моек — 10%
- 20 моек — 15%

### Расчёт цен

**Готовый package:**
```
originalPrice = washType.basePrice × washCount
price         = originalPrice × (1 − discountPercent / 100)
```

**Безлимит:**
- Цена фиксирована в плане и привязана к типу мойки.

**Персональный:**
```
singlePrice   = washType.basePrice + Σ(extra.price)
originalPrice = singlePrice × washCount
price         = originalPrice × (1 − volumeDiscount)
```

### UI/UX

- Главный экран клиента не изменяется.
- В боковом меню клиента добавляется пункт **«Абонементы»**.
- Пункт ведёт на экран-«хаб» с кнопкой «Купить абонемент» и списком активных абонементов.
- Покупка — пошаговый визард:
  1. Выбор типа: «Готовый» / «Персональный».
  2. Конфигурация плана.
  3. Экран подтверждения с итоговой ценой.
  4. Демо-оплата (кнопка без реального проведения платежа).
  5. Экран успеха.

## Архитектура

### Модель данных

#### Новая таблица: `subscription_plans`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int, PK | Автоинкремент |
| `code` | str, unique | Уникальный код плана |
| `name` | str | Название для клиента |
| `description` | str | Описание |
| `type` | str | `package` или `unlimited` |
| `washCount` | int | Количество моек (для `package`) |
| `unlimitedDays` | int | Длительность в днях (для `unlimited`) |
| `discountPercent` | int | Скидка на базовую цену типа мойки |
| `sortOrder` | int | Порядок отображения |
| `isActive` | bool | Активен ли план в каталоге |
| `createdAt` | datetime | — |
| `updatedAt` | datetime | — |

#### Расширение таблицы `subscriptions`

| Поле | Тип | Описание |
|------|-----|----------|
| `planId` | int, FK → subscription_plans | null для персональных и ручных админских |
| `price` | int | Итоговая цена абонемента |
| `originalPrice` | int | Цена без скидки |
| `selectedExtras` | JSON | Список id доп. услуг (для персональных) |
| `paymentStatus` | str | `demo_purchased`, `pending`, `paid` |

### API

#### Клиентские эндпоинты

- `GET /api/subscriptions/plans` — список активных планов для каталога.
- `POST /api/subscriptions/buy` — покупка абонемента.
  - Тело запроса зависит от типа:
    - Готовый: `{ planId, washTypeId }`
    - Персональный: `{ washTypeId, selectedExtras: string[], washCount }`
  - В ответе — созданный `Subscription`.
- `GET /api/subscriptions/my` — уже существует, возвращает активные абонементы текущего пользователя.

#### Админские эндпоинты

- `GET /api/admin/subscription-plans` — список всех планов.
- `POST /api/admin/subscription-plans` — создание плана.
- `PUT /api/admin/subscription-plans/{id}` — редактирование.
- `DELETE /api/admin/subscription-plans/{id}` — деактивация/удаление.

Существующий `POST /api/subscriptions/` (админ создаёт абонемент вручную) остаётся без изменений.

## Ограничения и допущения

- Реальная оплата не реализуется. Статус сразу `demo_purchased`.
- Планы управляются админом через отдельные админские эндпоинты. UI админки для управления планами выходит за рамки задачи и может быть добавлен позже (достаточно seed-данных и ручного CRUD через API).
- Безлимитный абонемент действует ровно 30 дней с момента покупки (`validUntil = createdAt + 30 дней`).
- При записи на мойку логика автоматического применения абонемента остаётся текущей: бэкенд ищет активный абонемент подходящего типа и устанавливает `paidPrice = 0`.

## Критерии успеха

- Клиент видит пункт «Абонементы» в боковом меню.
- Клиент может купить готовый абонемент с выбором типа мойки.
- Клиент может собрать персональный абонемент с допами и рассчитать цену.
- После покупки абонемент отображается в «Моих абонементах».
- При записи на мойку активный абонемент применяется автоматически.
- Все существующие тесты продолжают проходить.
