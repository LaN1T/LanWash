# Дизайн фазы 1: улучшение раздела смен

## Контекст

Экран `ShiftScheduleScreen` отображает таблицу «мойщики × дни недели». Сейчас ячейки мелкие, информации мало, массовые действия и копирование отсутствуют. Цель фазы 1 — быстрые улучшения UX без полной перестройки экрана.

## Scope фазы 1

### Включается

1. **Улучшенные карточки смены**
   - Увеличенные ячейки с чётким временем `10:00–18:00`
   - Продолжительность под временем
   - Цветовая индикация статуса: confirmed (primary), pending (warning), rejected (danger)
   - Пустая ячейка: светлый фон с иконкой `+` для editable, серый для readonly

2. **Копирование / вставка смен**
   - Долгое нажатие на ячейку смены открывает контекстное меню: «Копировать», «Удалить»
   - Долгое нажатие на ячейку без смены показывает «Вставить» (если в буфере есть смена)
   - Контекстное меню на имени мойщика: «Копировать неделю», «Вставить неделю», «Очистить неделю»
   - Визуальная индикация буфера: подсветка скопированной ячейки / строки
   - Буфер хранится в памяти состояния экрана (не persisted)

3. **Подсветка конфликтов**
   - Если у одного мойщика в один день несколько смен пересекаются — красная рамка ячейки
   - Если смена выходит за допустимый диапазон (например, начало до 06:00 или конец после 23:00) — предупреждающая иконка
   - Проверка выполняется локально на основе загруженных данных

4. **Панель «Кто на смене сейчас»**
   - В `AppBar` вместо текстового счётчика — горизонтальный список аватарок мойщиков на активных сменах
   - Тап по аватарке подсвечивает строку этого мойщика в таблице
   - Если на смене никого — скрыто

5. **Быстрые действия администратора**
   - Кнопки в `AppBar` (только для admin): «Одобрить все заявки», «Отклонить все заявки» для текущей недели
   - Применяются только к сменам со статусом `pending`
   - Подтверждение через AlertDialog

### Не включается

- Drag-and-drop между ячейками
- Шаблоны смен («утренняя / вечерняя / полный день»)
- Аналитика часов за период (месяц / квартал)
- Автозаполнение смен на основе загрузки боксов
- Изменение других экранов

## Архитектура

### UI

- `ShiftScheduleScreen` — основной экран, остаётся StatefulWidget
- `_ShiftCell` — выделить ячейку в отдельный widget с поддержкой долгого нажатия
- `_ShiftBuffer` — класс состояния буфера копирования
- `_OnDutyAvatars` — widget аватарок в AppBar
- `_BulkActionsMenu` — кнопки массовых действий админа

### State

```dart
class _ShiftScheduleScreenState extends State<ShiftScheduleScreen> {
  Shift? _copiedShift;        // одиночная скопированная смена
  List<Shift>? _copiedWeek;   // скопированная неделя мойщика
  int? _highlightedWasherId;  // для подсветки строки
  ...
}
```

### API

Используются существующие endpoints:
- `GET /api/shifts` — список смен за неделю
- `POST /api/shifts` — создание / обновление смены
- `DELETE /api/shifts/{id}` — удаление
- `PATCH /api/shifts/{id}/status` — изменение статуса (approve / reject)

Если PATCH-эндпоинт отсутствует, добавляется backend-роут в рамках фазы 1.

## Визуальные изменения

1. Ячейка смены:
   - Высота увеличена с 56 до 72
   - Время `11` → `13` font size, semibold
   - Длительность `9` font size
   - Радиус скругления `10` → `12`

2. Пустая ячейка:
   - Фон `adaptiveBgPage`
   - Иконка `+` цвета `adaptiveTextTertiary`

3. Конфликт:
   - Красная рамка `2px`
   - Иконка `Icons.warning_amber_rounded` размером 14 в углу

## Тестирование

- Виджет-тесты для `_ShiftCell`:
  - отображение confirmed/pending/rejected
  - долгое нажатие вызывает меню
  - конфликт подсвечивает рамку
- Интеграционный тест:
  - копирование смены и вставка в другой день
  - массовое одобрение pending-смен

## Критерии приёмки

- [ ] Админ видит кнопки массовых действий
- [ ] Мойщик может копировать/вставлять только свои смены
- [ ] Конфликты визуально заметны
- [ ] Буфер очищается при закрытии экрана
- [ ] Все существующие тесты проходят
