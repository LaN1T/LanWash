# Доделка отзывов + перенос кнопки «Записаться снова»

## Goal

В истории записей на экране `MyBookingsScreen` завершённой записи (`completed`) снизу должна предлагаться кнопка «Оставить отзыв», а кнопка «Записаться снова» переезжает в верхнюю часть карточки. Отменённые записи (`cancelled`) сохраняют кнопку «Записаться снова» внизу. Если отзыв уже оставлен — кнопка внизу становится неактивной серой надписью «Отзыв оставлен».

## Scope

- Изменения только в Flutter UI.
- Backend используется существующий (`GET /api/reviews/has-review`, `POST /api/reviews`).
- Экран `ReviewCreateScreen` уже умеет создавать отзыв по `appointmentId`.

## UI Changes

### Карточка в `MyBookingsScreen` (`_BookingsList`)

1. **Верхняя строка (header row):**
   - Справа от информации о записи добавить иконку-обновление `Icons.refresh` (Tooltip «Записаться снова»).
   - Тап по ней открывает `BookingWizardScreen(templateAppointment: a)`.
   - Показывается для всех записей во вкладке «История» (`showBookAgain == true`).

2. **Нижняя часть карточки (`showBookAgain == true`):**
   - Если `a.status == 'completed'`:
     - Асинхронно проверяем `hasReviewForAppointment(a.id)`.
     - Если отзыва нет — `OutlinedButton.icon` с иконкой `Icons.rate_review` и текстом «Оставить отзыв».
       - Тап открывает `ReviewCreateScreen(appointmentId: a.id)`.
       - После возврата обновляем проверку (можно через `setState` на уровне карточки).
     - Если отзыв есть — неактивный `OutlinedButton.icon` (серый цвет `AppStyles.adaptiveTextSecondary`) с текстом «Отзыв оставлен».
   - Если `a.status == 'cancelled'`:
     - Оставляем текущую кнопку «Записаться снова» внизу (она больше не дублируется сверху, только иконка сверху).
   - Других статусов во вкладке истории не предусмотрено.

### `ReviewCreateScreen`

- Может принимать `Appointment?` в дополнение к `appointmentId`, чтобы показывать название типа мойки в заголовке/подсказке.
- После успешной отправки возвращает `true` через `Navigator.pop(context, true)`, чтобы вызывающая сторона могла обновить состояние.

## State handling

- Проверка наличия отзыва выполняется внутри карточки через `FutureBuilder` или `StatefulBuilder`, чтобы не грузить весь список синхронно.
- Ошибка проверки трактуется как «отзыва нет» — показываем кнопку «Оставить отзыв».

## Edge cases

- Нет сети при проверке — кнопка «Оставить отзыв» остаётся активной.
- Пользователь закрыл экран отзыва без отправки — состояние не меняется.
- Повторный отзыв блокируется на backend (unique constraint), UI просто покажет неактивную надпись после первого успешного создания.
